Dashboard
Overview summary — pending claim count, active policies, recent activity feed. Built from /api/admin/audit-logs/recent and /api/admin/policies/summary.
All Claims
Full claim list with status filters (DRAFT, SUBMITTED, UNDER_REVIEW, APPROVED, REJECTED). Tap any row to see claim detail with downloadable documents.
GET /api/admin/claims GET /api/admin/claims/{id} GET /api/claims/{id}/download/claim-form GET /api/claims/{id}/download/aadhaar GET /api/claims/{id}/download/evidence
Under Review
Action queue — claims awaiting admin decision. Approve or reject with remarks directly from this view. Most time-sensitive screen for admin workflow.
GET /api/admin/claims/under-review PUT /api/admin/claims/{id}/review PUT /api/admin/claims/{id}/approve PUT /api/admin/claims/{id}/reject
All Policies
Paginated list of all customer policies. Admin can cancel a policy with optional reason, update status, and view per-policy premiums. Summary KPIs available.
GET /api/policies/admin/all GET /api/admin/policies/{id} PUT /api/admin/policies/{id}/cancel PUT /api/policies/admin/{id}/status GET /api/policies/admin/summary
Policy Types (Product Catalog)
Create, edit, and discontinue insurance product types. Supports filtering by insurance category. This is the only full-CRUD section exclusively for admins.
GET /api/policy-types/all POST /api/policy-types PUT /api/policy-types/{id} DELETE /api/policy-types/{id} GET /api/policy-types/category/{cat}
All Users
Paginated user list with full profile lookup. Admin can view, edit, delete users and manage their addresses. Sortable by userId, name, etc.
GET /user/getAll GET /user/getInfo/{id} PUT /user/update/{id} DELETE /user/delete/{id} GET /user/getAddress/{id}
Payments
Look up any payment by transaction ID. Useful for resolving disputes or confirming premium collection. Health check indicator for payment service status.
GET /api/payments/transaction/{txnId} GET /api/payments/health-check
Audit Logs
Full admin action history. Filter by date range, look up history for a specific claim or policy, or view latest N entries. Essential for compliance and debugging.
GET /api/admin/audit-logs GET /api/admin/audit-logs/recent GET /api/admin/audit-logs/{entity}/{id} GET /api/admin/audit-logs/range
